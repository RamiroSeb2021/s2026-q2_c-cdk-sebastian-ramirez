# AWS CDK Hello World Lambda

This project defines and deploys a simple AWS Lambda function using AWS CDK with TypeScript. The stack creates a Lambda function that returns `Hello CDK!` and exposes it through a public Lambda Function URL.

## What was built

| Resource | Purpose |
|----------|---------|
| Lambda function | Runs an inline Node.js 20 handler that returns a JSON response. |
| Lambda Function URL | Provides an HTTPS endpoint to invoke the function without API Gateway. |
| CloudFormation output | Prints the deployed Function URL after `cdk deploy`. |
| Lambda service role | Allows Lambda to run and write logs to CloudWatch. |

## Project structure

```text
bin/    CDK app entry point
lib/    CDK stack definition
test/   Jest tests generated by the CDK template
```

The main infrastructure code lives in:

```text
lib/s2026-q2_c-cdk-sebastian-ramirez-stack.ts
```

## Prerequisites

- Node.js 22 or compatible
- npm
- AWS CLI configured with valid credentials
- AWS CDK CLI through `npx cdk`

If you use temporary AWS credentials, make sure the `aws_session_token` is current before deploying.

You can verify the active AWS account with:

```bash
aws sts get-caller-identity --query "Account" --output text
```

## Install dependencies

```bash
npm install
```

## Deploy

Bootstrap the AWS environment once per account and region:

```bash
npx cdk bootstrap
```

Deploy the stack:

```bash
npx cdk deploy
```

During deployment, CDK asks for approval because the stack creates IAM permissions for the Lambda service role and Function URL invocation.

After the deployment finishes, CDK prints the `myFunctionUrlOutput` value. Open that URL in a browser or call it with `curl` to test the Lambda.

## Useful commands

| Command | Description |
|---------|-------------|
| `npm run build` | Compile TypeScript to JavaScript. |
| `npm run watch` | Watch for changes and compile automatically. |
| `npm run test` | Run Jest tests. |
| `npx cdk synth` | Generate the CloudFormation template locally. |
| `npx cdk diff` | Compare local infrastructure with the deployed stack. |
| `npx cdk deploy` | Deploy the stack to the configured AWS account and region. |
| `npx cdk destroy` | Delete the deployed stack and its resources. |

## Cleanup

To avoid leaving AWS resources running after the exercise, destroy the stack when you no longer need it:

```bash
npx cdk destroy
```
